Functional Programming

The building block of a functional programming language is the function.

First Class Functions

Can be passed into a function or returned from a function

Higher Order Functions

Functions that take another function as a parameter

  • (inc 1) ;=> 2
  • (map inc [1 2 3]) ;=> [2 3 4]
  • [1, 2, 3].map(&:next) ;=> [2, 3, 4]

Immutable Data

Any mutation of data is returned as a new value instead of mutating the original value.

  • (def a 1) #'user/a
  • (+ a 1) ;=> 2
  • a ;=> 1

Pure functions

  • Values In -> Values Out
  • No Side Effects

Referential Transparency

  • A function can be substituted with it's value without affecting the program.

Persistent Data Structures

1: (def base-list [1 2 3])
2: (def a-list (cons "a" base-list))
3: (def b-list (cons "b" base-list))
4: 
5: (= (rest a-list) (rest b-list))
6: (identical? (rest a-list) (rest b-list))
