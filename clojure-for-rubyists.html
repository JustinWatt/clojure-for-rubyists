Tour of Clojure

Clojure

A LISP that targets the JVM

Syntax Primer

  • add 1, 2
  • add(1, 2)
  • (+ 1 2)
  • (function arg1 arg2)

Functional Programming

First Class Functions

Can be passed into a function or returned from a function

Higher Order Functions

Functions that take another function as a parameter

  • (inc 1) ;=> 2
  • (map inc [1 2 3]) ;=> [2 3 4]
  • [1, 2, 3].map(&:next) ;=> [2, 3, 4]

Immutable Data

Instead of mutating a value in place, a new copy of the structure is created with the mutation applied

  • (def a 1) #'user/a
  • (+ a 1) ;=> 2
  • a ;=> 1

Persistent Data Structures

  • (def base-list [1 2 3])
  • (def a-list (cons "a" base-list))
  • (def b-list (cons "b" base-list))
  • (identical? (rest a-list) (rest b-list)) ;=> true

Pure functions

  • Values In -> Values Out
  • No Side Effects

Referential Transparency

  • A function can be substituted with it's value without affecting the program.

s-expressions

  • The primary unit of computation in Clojure is the s-expression.
  • (function-name arg1 arg2 arg3)
  • Functions are applied and the expressions reduce down to a single value
  • Expressions are nested to achieve more complex effects
    • (reduce + (filter odd? (map inc [1 2 3 4])))
    • (reduce + (filter odd? [2 3 4 5]))
    • (reduce + [3 5])
    • 8

Transducers

(->> [1 2 3 4]
     (map inc)
     (filter odd?) 
     (reduce +))

Data Structures

  • List: '(1 2 3 4)
    • Items are added to the front of a list
  • Vector: ([1 2 3 4] 0)
    • ;=> 1
  • Map: {:id 1 :name "Cen Cal Ruby"}
    • (:name {:id 1 :name "Cen Cal Ruby" )}
    • ;=> "Cen Cal Ruby"
  • Set: #{ :this :is :a :set }
